# 世界杯 2026 观赛 App — 设计规格

- **日期:** 2026-06-11
- **状态:** 已批准设计, 待写实现计划
- **原型:** `prototype/index.html`(可点击, 已确认)

## 1. 目标

移动 App, 实时查看 2026 美加墨世界杯赛程、实况、历史结果、积分/对阵, 并展示中国体彩竞彩赔率。

## 2. 范围 (MVP)

**做:**
- 仅 FIFA World Cup 2026
- 5 个功能页: 赛程 / 实况 / 结果 / 积分榜+对阵图 / 竞彩赔率
- 实况自动轮询(30–60 秒)+ 全局下拉刷新
- 后端代理(持密钥、缓存、限流、抓取赔率)

**不做(以后):**
- 推送通知 / 多赛事 / 用户登录与收藏 / 真正的 WebSocket 推送
- 任何投注功能(仅展示赔率)

## 3. 架构

```
┌──────────────┐      ┌────────────────────┐      ┌────────────────┐
│  Flutter App │─────▶│   后端代理 (FastAPI) │─────▶│  API-Football  │
│ (Android/iOS)│◀────│  缓存 + 限流 + 密钥   │◀────│  (api-sports)  │
└──────────────┘      │                    │      └────────────────┘
   每30-60秒轮询        │  爬虫定时抓取赔率     │─────▶ 500.com / okooo
   全局下拉刷新         └────────────────────┘        (竞彩赔率源)
```

代理把 App 的大量请求合并成少量上游调用。N 个用户看同一场实况 = 每 30 秒只打 1 次 API-Football, 保住免费额度(100 次/天)。

**技术选型:**
- App: **Flutter**(Android + iOS 单代码库; Windows 上先出 Android, iOS 后续需 Mac 构建)
- 代理: **FastAPI (Python)**
- 部署: 开发期本地运行; 上线后免费部署(Render / Railway / Fly.io)

## 4. 组件

### 4.1 App (Flutter)
- 底部 5 标签导航
- 各页通过 HTTP 调用代理, 不直连上游, 不内置任何密钥
- 本地缓存(SQLite 或 shared_preferences + 内存): 重开秒显示、弱网可用
- 实况页用定时器(`Timer.periodic`)轮询; 离开页面即停, 省电省流量

页面:
| 页 | 内容 |
|----|------|
| 赛程 | 按日期分组的未开赛列表; 筛选 chip(小组/16强/8强/关注) |
| 实况 | 重点直播(大比分+分钟+事件时间轴) + 其他进行中比赛 |
| 结果 | 已完赛历史比分(含点球), 按日期分组 |
| 积分/对阵 | 小组积分表(出线区高亮) ↔ 横向滑动淘汰赛对阵图 |
| 赔率 | 选比赛 → 胜平负/让球胜平负/比分/总进球/半全场 5 玩法 |

### 4.2 代理 (FastAPI)
端点(均返回 App 友好的精简 JSON):
- `GET /fixtures` — 赛程
- `GET /live` — 进行中比赛 + 事件
- `GET /results` — 已完赛
- `GET /standings` — 小组积分
- `GET /bracket` — 淘汰赛对阵
- `GET /odds?matchId=` — 竞彩赔率

机制:
- API 密钥放环境变量, 绝不下发到 App
- 内存缓存带 TTL: 实况 30s, 赛程 1h, 结果 6h, 积分 1h, 赔率 5min
- 上游 429/超时 → 返回上次缓存 + 标记 `stale:true`
- 赔率: 后台定时任务抓取第三方并入缓存; 抓取失败该场返回 `odds:null`

## 5. 数据流(实况举例)

App 实况页定时器触发 → `GET /live` → 代理查缓存 → 新鲜则返回缓存 : 否则调 API-Football, 解析, 存缓存, 返回 → App 比对差异、重绘比分与时间轴。

## 6. 数据源 & 字段映射

| 数据 | 源 | 关键字段 |
|------|----|---------|
| 赛程/实况/结果 | API-Football `/fixtures` | 队、比分、状态、分钟、场地、阶段 |
| 事件(进球/牌/换人) | API-Football `/fixtures/events` | 分钟、类型、球员、队 |
| 积分 | API-Football `/standings` | 名次、赛/胜/平/负、净、分 |
| 对阵 | 由淘汰赛 fixtures 推导 | 轮次、对阵、比分 |
| 竞彩赔率 | 抓取 500.com / okooo | 胜平负、让球、比分、总进球、半全场 |

## 7. 错误处理

- 网络失败 → 显示缓存 + "数据可能过时"提示, 提供重试
- 上游限额(429) → 代理降频, App 显示上次数据
- 赔率抓取失败 → 该场显示"暂无赔率", 其他功能不受影响
- 空状态(当日无比赛)→ 友好占位图

## 8. 测试策略

- 代理: 单元测试缓存 TTL 逻辑、上游错误降级、字段映射(用录制的样例响应, 不打真实 API)
- 抓取器: 用保存的 HTML 样本做解析测试(隔离网站改版风险)
- App: widget 测试各页渲染; 用 mock 代理响应测交互(切页、轮询、下拉刷新)

## 9. 已知风险

1. **抓取稳定性** — 第三方站点改版/反爬会断数据。缓解: 解析器隔离 + 失败降级 + 解析测试。
2. **商店合规** — 含赔率内容在 iOS App Store / 国内安卓商店审核严, 公开发布可能被拒或需资质。MVP 定位自用/学习。
3. **iOS 构建** — Windows 无法出 iOS 包; 需 Mac 或云构建(Codemagic 等)。MVP 先 Android。
4. **API 免费额度** — 100 次/天; 靠代理缓存撑住, 高并发需升级套餐。

## 10. 合规声明

App 仅**展示**公开赛事信息与赔率, 不提供任何投注功能。界面含"理性看球、远离非法赌博"提示。
